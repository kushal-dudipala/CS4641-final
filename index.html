Introduction/Background

Predicting house prices is critical to the real estate industry, as it provides valuable insight for buyers, sellers, and even investors. Since we have access to housing data, we want to build a model that takes into account various home characteristics, such as square footage and home type, for accurate and reliable home price predictions.

Studies such as by Kok et al. (2017) utilized a linear regression model to predict home prices with features including house size, number of rooms, and location. The model was proven accurate based on the given training dataset, and the results were easy to interpret.

We will be using the Ames Housing Dataset, an alternative to the popular Boston Dataset found in James et al. (2018), but with more features and observations. This dataset can be found at the following link: Ames Housing Dataset on Kaggle

Problem Definition

Problem

The problem we are addressing is the challenge of accurately predicting house prices located in Ames, Iowa. Traditional methods often rely on simple models or manual predictions which don’t capture the more complex relationships between predictive variables. This leads buyers, sellers, and investors to make suboptimal decisions because of a lack of accurate information.

Motivation

By accurately automating house price prediction, it would help homebuyers, sellers, real estate agents, and investors make informed decisions.

Methods

Model information:

  • Preprocessing:
    • Removing Columns with Excessive Missing Values
    • Imputation of Missing Values
    • Encoding Categorical Variables
    • Combining training and testing sets
  • Model:
    • LASSO Regression

Our methods for data processing and the use of the Lasso Regression model are well suited to our project due to the nature of our data. The preprocessing steps ensure consistency through our dataset by removing data columns that have too much information missing and then encoding categories appropriately. Lasso regression is then used for its ability to perform feature selection and to prevent overfitting through regularization. The tuning of the hyperparameter along with the subsequent validation of the model, gives an accurate and generalizable prediction model.

This is done by first loading both the training and testing sets and adding a TrainFlag to distinguish between them and concatenating it into one data frame, creating a consistent preprocessing process. Afterwards we remove columns with excessive missing values, our threshold sits at 50%. SimpleImputer is used to handle missing data values, with all the data turning into a numerical format to use regression models on. The median was used for imputation because it is effective against outliers and any skewed distributions that we could come across, such as property sizes or prices. We also used mode imputation for categorical data to simplify the process when missing values are mostly random.

Lasso regression is used to reduce overfitting and to select the most important features of the data. This algorithm uses L1 regularization and the lasso model with a specific alpha value to find the optimal values. The alpha values tested range from [0.0001 – 1.0] to find the optimal regularization strength that will minimize the prediction error. The model then is trained with the data and the best alpha value is found for the prediction to be as accurate as possible. This lasso regression model was used to basically function as a feature selection since our data set has so many features and complexity needed to be reduced. The regularization used also works to discourage the model from overfitting the noise in the data. The linear nature also helps with the interpretability of the data.
